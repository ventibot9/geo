# GEO 平台架构设计

## GitHub仓库
https://github.com/ventibot9/geo.git

## 项目概述
GEO（生成式引擎优化）SaaS平台 - 帮企业优化内容结构，提高AI引用率

## 核心功能
1. 自动扫描：定期扫描企业网站/内容库，分析AI友好度
2. AI改写：自动优化内容结构，生成AI易读格式
3. 引用监控：追踪各大AI引擎引用情况
4. 数据报表：可视化GEO优化效果

## 技术架构

### 整体架构
```
┌─────────────────────────────────────────────────┐
│                前端管理界面                      │

└─────────────────────────────────────────────────┘
                       ↓
┌─────────────────────────────────────────────────┐
│              后端 API 网关                       │

└─────────────────────────────────────────────────┘
           ↓         ↓         ↓         ↓
    ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐
    │内容扫描器│ │AI改写服务│ │引用监控器│ │数据分析器│

           ↓         ↓         ↓         ↓
    ┌─────────────────────────────────────────────┐

```

### 模块分工
1. **前端** (`frontend/`)
   - 企业用户管理界面
   - 内容扫描结果展示
   - AI改写编辑器
   - 数据分析仪表盘
   - 引用监控面板

2. **后端API** (`backend/`)
   - RESTful API
   - 用户认证授权（JWT）
   - 企业多租户管理
   - 任务调度（Celery/Redis）
   - Webhook通知

3. **内容扫描器** (`scanner/`)
   - 网站爬虫
   - 内容结构分析
   - AI友好度评分
   - 优化建议生成

4. **AI改写服务** (`ai-service/`)
   - 接入大模型API（OpenAI/Anthropic/本地模型）
   - 内容结构优化
   - Markdown/API文档格式转换
   - 批量改写

5. **引用监控器** (`monitor/`)
   - 监控ChatGPT/Claude/文心一言等
   - 追踪企业内容引用
   - 曝光量统计
   - 趋势分析

6. **数据分析器** (集成在后端)
   - GEO优化效果统计
   - 引用趋势报表
   - ROI分析

### 数据模型
```
企业 (Enterprise)
  ├── 用户 (User)
  ├── 网站配置 (SiteConfig)
  │   ├── 扫描任务 (ScanTask)
  │   │   ├── 内容页面 (ContentPage)
  │   │   │   ├── AI友好度评分 (AIReadabilityScore)
  │   │   │   ├── 优化建议 (OptimizationSuggestion)
  │   │   │   └── AI改写记录 (AIRewriteRecord)
  │   │   └── 引用数据 (CitationData)
  │   └── 监控任务 (MonitorTask)
  └── 订阅计划 (SubscriptionPlan)
```

### 技术栈
- **前端**: React 18 + TypeScript + Ant Design + Vite
- **后端**: Node.js 20 + Express + TypeScript + Prisma
- **数据库**: PostgreSQL 15 + Redis 7
- **任务队列**: Bull (基于Redis)
- **AI模型**: OpenAI API / Anthropic API / 本地Llama
- **爬虫**: Puppeteer + Cheerio
- **监控**: Prometheus + Grafana
