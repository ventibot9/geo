## 1. 项目概述

### 1.1 项目定位
GEO（Generative Engine Optimization）SaaS 平台，面向企业客户提供：
- 内容结构化分析
- AI 友好度优化
- 智能内容改写
- AI 引用监控
- 数据分析报表

### 1.2 目标用户
- **企业客户**: 需要优化网站内容以获得更高 AI 引用率
- **运营人员**: 管理扫描任务、查看优化效果
- **内容团队**: 使用 AI 改写工具优化内容

### 1.3 核心价值
1. **提高曝光**: 通过优化内容结构，让 AI 引擎更容易引用
2. **节省时间**: 自动化扫描 + AI 改写，减少人工工作量
3. **数据驱动**: 实时监控引用情况，量化 GEO 优化效果

### 2.2 技术栈选择

#### 前端
- **框架**: React 18.2+ (函数组件 + Hooks)
- **语言**: TypeScript 5.3+ (类型安全)
- **构建工具**: Vite 5.0+ (快速 HMR)
- **UI 库**: Ant Design 5.12+ (企业级组件)
- **状态管理**: Zustand 4.4+ (轻量级)
- **路由**: React Router 6.22+ (基于文件路由)
- **HTTP 客户端**: Axios 1.6+ (拦截器 + 错误处理)
- **图表**: Canvas / ECharts (数据分析可视化)

**选择理由**:
- React: 生态成熟，开发者熟悉度高
- Vite: 构建速度比 Webpack 快 10-20 倍
- Ant Design: 企业级 UI 组件，开箱即用
- Zustand: 比 Redux 轻量 10 倍，学习曲线低

#### 后端
- **运行时**: Node.js 20 LTS (ESM + Fetch API)
- **框架**: Express 4.18+ (最流行的 Node.js 框架)
- **语言**: TypeScript 5.3+ (端到端类型安全)
- **ORM**: Prisma 5.7+ (类型安全的数据库客户端)
- **队列**: Bull 4.16+ (基于 Redis 的任务队列)
- **认证**: jsonwebtoken 9.0+ (JWT 令牌)
- **密码加密**: bcryptjs 2.4+ (哈希加盐)

**选择理由**:
- Express: 灵活，中间件生态丰富
- Prisma: 类型安全，迁移简单
- Bull: Redis 原生支持，性能优秀

#### 微服务 (Python)
- **运行时**: Python 3.11+ (PEP 594)
- **Web 框架**:
  - Scanner: 无 (CLI 工具 + 任务队列消费者)
  - AI Service: FastAPI 0.104+ (高性能异步框架)
  - Monitor: 无 (后台任务 + API)
- **爬虫**: Puppeteer (支持 JS 渲染)
- **HTML 解析**: BeautifulSoup 4.12+ (容错性强)
- **数据库**: SQLAlchemy 2.0+ (Python ORM)
- **任务调度**: APScheduler 3.10+ (CRON + 间隔任务)
- **HTTP 客户端**: httpx 0.26+ (异步请求)

**选择理由**:
- Python 3.11: 性能提升 15-30%，更好的类型提示
- FastAPI: 性能比 Flask 快 5-10 倍
- Puppeteer: 支持单页应用 (SPA) 和动态内容

#### 数据库
- **主数据库**: PostgreSQL 15+ (关系型 + JSONB)
- **缓存**: Redis 7+ (内存数据库)
- **连接池**: pg-pool 3.0+ (连接复用)

**选择理由**:
- PostgreSQL: 支持 JSONB (适合半结构化数据)、全文搜索、窗口函数
- Redis: 性能优秀 (10 万 QPS+)、支持多种数据结构

## 3. 数据流设计

### 3.1 用户认证流程
```
1. 用户输入邮箱/密码
   ↓
2. Frontend → Backend API: POST /api/auth/login
   ↓
3. Backend 验证密码 (bcrypt)
   ↓
4. 生成 JWT Token (有效期 7 天)
   ↓
5. 返回 Token + 用户信息
   ↓
6. Frontend 存储 Token (LocalStorage + Zustand)
   ↓
7. 后续请求自动携带 Token (Axios 拦截器)
   ↓
8. Backend 验证 Token (中间件)
   ↓
9. 通过: 返回数据
   失败: 返回 401 → 跳转登录
```

### 3.2 扫描任务流程
```
1. 用户在 Frontend 配置扫描任务
   ↓
2. POST /api/scans/create → 创建 ScanTask
   ↓
3. 加入 Bull 队列 (scan-queue)
   ↓
4. Scanner Service 消费任务
   ↓
5. 爬取网站内容 (Puppeteer)
   ↓
6. 分析 AI 友好度 (5 维度评分)
   ↓
7. 生成扫描报告 (JSON/Markdown)
   ↓
8. 更新 ScanTask 状态为 "completed"
   ↓
9. 通过 WebSocket 推送更新到 Frontend
```

### 3.3 AI 改写流程
```
1. 用户选择内容 → Frontend 编辑器
   ↓
2. POST /api/ai/rewrite → 创建 RewriteRecord
   ↓
3. 加入 Bull 队列 (rewrite-queue)
   ↓
4. AI Service 消费任务
   ↓
5. 选择大模型 (GPT-4 / Claude / Llama)
   ↓
6. 调用 API (OpenAI / Anthropic / 本地)
   ↓
7. 获取结构化内容 (Markdown + 表格)
   ↓
8. 评估改写质量 (多维度评分)
   ↓
9. 更新 RewriteRecord 状态为 "completed"
   ↓
10. Frontend 获取结果并展示对比视图
```

### 3.4 引用监控流程
```
1. 定时任务触发 (APScheduler, 每 6 小时)
   ↓
2. Monitor Service 获取企业关键词
   ↓
3. 生成查询 (模板 + 关键词组合)
   ↓
4. 调用 AI 引擎 API
   - ChatGPT: OpenAI API
   - Claude: Anthropic API
   - 文心一言: 搜索方式
   ↓
5. 提取引用数据 (关键词匹配)
   ↓
6. 计算置信度评分 (0-1)
   ↓
7. 存储到 CitationRecord
   ↓
8. 聚合统计 (KeywordStats 表)
   ↓
9. 生成监控报告 (JSON / 文本)
```

## 4. 安全设计

### 4.1 认证与授权
- **认证方式**: JWT Bearer Token
- **Token 有效期**: 7 天 (可配置)
- **密码策略**:
  - 最小长度: 8 字符
  - 必须包含: 字母 + 数字
- **刷新机制**: 自动刷新 (Token 有效期前 1 天)

### 4.2 企业多租户隔离
- **数据库隔离**: 所有查询自动注入 `enterprise_id`
- **文件隔离**: 按企业 ID 分目录存储
- **队列隔离**: 任务消息包含 `enterprise_id`
- **缓存隔离**: Redis Key 前缀企业 ID

### 4.3 API 安全
- **HTTPS**: 强制使用 HTTPS (Nginx 层)
- **CORS**: 仅允许白名单域名
- **Rate Limiting**:
  - 认证端点: 5 次 / 分钟
  - 普通端点: 100 次 / 分钟
- **输入验证**:
  - SQL 注入防护 (Prisma 参数化查询)
  - XSS 防护 (前端转义 + CSP)
- **敏感数据加密**:
  - 密码: bcrypt (cost factor 10)
  - API 密钥: 环境变量 (不进代码库)

### 6.2 服务依赖
```
Frontend → Backend (REST API)
Backend → PostgreSQL (数据库)
Backend → Redis (队列 + 缓存)
Backend → Scanner (任务消息)
Backend → AI Service (任务消息)
Backend → Monitor (任务消息)
Scanner → PostgreSQL (写入扫描结果)
AI Service → PostgreSQL (写入改写记录)
Monitor → PostgreSQL (写入引用数据)
```

### 6.3 健康检查
每个服务暴露 `/health` 端点：
- **Frontend**: 返回静态资源状态
- **Backend**: 检查数据库 + Redis 连接
- **Scanner**: 检查爬虫能力
- **AI Service**: 检查 API 密钥可用性
- **Monitor**: 检查调度器状态

## 7. 监控与日志

### 7.1 日志策略
- **日志级别**: DEBUG / INFO / WARN / ERROR
- **日志格式**: JSON (结构化日志)
- **日志位置**:
  - 开发环境: stdout (Docker logs)
  - 生产环境: 集中式日志系统 (ELK)
- **日志轮转**: 每天一个文件，保留 30 天

### 7.2 监控指标
- **系统指标**:
  - CPU 使用率
  - 内存使用率
  - 磁盘 I/O
  - 网络流量
- **应用指标**:
  - API 请求量 (QPS)
  - API 响应时间 (P50/P95/P99)
  - 任务队列长度
  - 错误率
- **业务指标**:
  - 活跃企业数
  - 扫描任务成功率
  - AI 改写质量平均分
  - 引用监控覆盖率

### 7.3 告警规则
- **系统告警**:
  - CPU > 80% (持续 5 分钟)
  - 内存 > 90% (持续 5 分钟)
  - 磁盘 > 85%
- **应用告警**:
  - API 错误率 > 5%
  - 任务队列长度 > 1000
  - 数据库连接失败 > 10 次 / 分钟
- **业务告警**:
  - 扫描任务失败率 > 20%
  - AI 改写质量分 < 60 (持续 1 小时)

## 8. 扩展性设计

### 8.1 水平扩展
- **Frontend**: 无状态，可多实例部署 (Nginx 负载均衡)

- **增加资源**: Docker Compose 调整 CPU / 内存限制
- **数据库优化**: 索引优化 + 查询优化
